# データモデル規約

商流図鑑のグラフデータは `data/schema.json`(JSON Schema)に従う。
業界ごとに1ファイル: `data/industries/<industry_id>.json`。

## 構造

- `meta` — 業界ID・名称・バージョン・更新日・編集注記
- `layers` — 業界ごとの階層定義(スキーマ側では固定しない。横展開時は業界ごとに宣言)
- `nodes` — プレイヤー(役割)。セマンティックズームの「引き」では `role` のみ、「寄り」で `companies` を表示
- `edges` — フロー。マップUIのフィルタ(すべて / モノ・サービス / カネCAPEX / カネOPEX)は `flow_type` で切り替える

## エッジの向きのルール

| flow_type | 向き | 例 |
|---|---|---|
| `goods` | 提供者 → 受領者 | 電力会社 → DC事業者(電力供給) |
| `capex` | 支払者 → 受領者(一時金) | DC事業者 → ゼネコン(建設工事費) |
| `opex` | 支払者 → 受領者(継続) | テナント → DC事業者(利用料) |

カネは常に「カネが流れる向き」。goodsとカネのペアは逆向きの2本のエッジで表現する
(すべてのペアを両方向で持つ必要はなく、可視化価値の高い方を優先)。

判断に迷う場合: 一時的な投資・調達は `capex`、継続的な支払い(利用料・仕入・保守・利息・税)は `opex`。

## 地図レイアウトのヒント (`node.map`)

- `ring`: 0 = 中央(DCサイト=コロケーション事業者)。外側ほど大きい整数
- `angle`: 北 = 0、時計回りの度数。おおまかな方角の目安で、UI側の自動レイアウトが微調整してよい
- 方角の割当(datacenter): 北=需要家、東=電力・通信、南=建設・設備、西=IT機器、北西=土地・資金

## 出典ポリシー(CLAUDE.md準拠)

- 一次情報のみ: 官公庁統計、IR・有報、JDCC、プレスリリース、企業サイト、自己登録データ
- 他社レポート・業界地図(xenoBrain・SPEEDA等)は転載・取込禁止
- `sources[].note` に「差し替え予定」とあるものは、構造レベルの仮出典。
  個別の一次情報URL(プレスリリース該当ページ等)への差し替えがv1完了の条件
- 金額感(`amount_note`)は公表情報ベースの規模表現のみ。個社の非公開情報・推計マージン率は書かない

## サブくくり(segments)と業界間リンク

- `node.segments` でノード内のサブくくりを宣言できる(例: 需要家ノード内の「自動車」「サーバー」)
- 企業は `company.segment` でサブくくりに所属する
- `segments[].related_industry` または `node.related_industry` に業界IDを書くと、
  地図・パネルからその業界の地図へ「潜る」ポータルが表示される

## 財務概算値(financials)・従業員数・上場情報

- `company.financials` に売上高・時価総額を**億円単位の概算値**で持てる(パネルの並び替えに使用)
- `company.employees` は連結従業員数の概算、`company.listing` は `{market, code}`
  (例: `{"market": "東証プライム", "code": "7203"}`。非上場は `{"market": "非上場(○○傘下)"}`)
- v1の値は公開情報ベースの規模感であり、`note` に「概算」と換算条件を必ず明記する
- 外貨は概算レート(例: 1ドル=150円)で換算し、noteに記載
- **IR・有報の確定値への差し替えがv1完了の条件**(出典ポリシーと同じ扱い)
- 数値が入れられない場合(非上場等)は note のみでもよい

## 業界の親子関係とマップ形式

- `meta.parent_industry` で親業界を宣言(例: auto_parts → automotive)。
  市場が大きい子業界は独立マップとして成立させる(自動車部品が第1号)
- `meta.map_style`:
  - `flow`(既定): 商流図。エッジ必須、孤立ノード禁止
  - `category`: カオスマップ型。商流が図示しにくい業界(コンサル等)は
    ラベル分類のノードだけで構成してよい(エッジ空配列可)
- 商流が複雑すぎる場合の妥協則: **大雑把な供給カテゴリでノードを括る**
  (自動車部品=パーツ別、コンサル=系譜・領域別)

## 全上場企業カバーの運用(達成済み・維持フェーズ)

最終目標は日本の全上場企業(約3,900社)がいずれかのマップに載っている状態。

1. 親業界は**時価総額の大きい業種から**追加する(index.jsonの `planned` が待ち行列)
2. 精緻な商流図が描けない業界も、まず `category` 型で企業を収容 → 後からflow型に昇格
3. 網羅の照合はJPX公表の上場銘柄一覧(東証33業種コード付き・月次更新)と突き合わせる。
   「どの上場企業がまだどのマップにも居ないか」を出すスクリプトを scripts/ に追加予定

## 掲載の中立性

- `companies[].plan`(掲載プラン)と `hiring`(採用バッジ)は情報量・機能の差のみに使う
- ノードの位置(`map`)・企業の掲載順は編集方針で決め、課金で変えない

## 検証

エッジの `from`/`to` が実在ノードを指すか等の整合性チェック:

```bash
node scripts/validate.mjs
```
